# 💬 通用会话启动指令 (长期使用版)

## 🎯 指令目的
这是一个通用的、可以长期使用的会话启动指令模板，适用于任何时候需要新开会话继续房源数据分析系统项目工作的场景。

## 🔥 通用启动指令 (复制使用)

```
由于对话长度限制，我将在新会话中继续工作。请帮我快速同步房源数据分析系统项目的当前状态：

1. 读取最新项目状态：
   - 查找并读取最新的项目状态文档(CURRENT_PROJECT_STATUS_*.md)
   - 查找并读取最新的课程要求对照文档(项目开发进度对照文档_*.md)
   - 查找并读取最新的文档索引(📚_项目文档索引_*.md)
   - 查找并读取最新的启动配置指南(🚀_项目启动配置_*.md)

2. 检查系统运行状态：
   - 使用list-processes查看当前运行的服务进程
   - 确认Django服务器(通常端口8000)运行状态
   - 确认MongoDB数据库(通常端口27017)运行状态  
   - 确认Elasticsearch集群(通常端口9200)运行状态
   - 验证数据库连接和数据完整性

3. 理解项目背景：
   - 这是一个数据采集课程设计项目，基于房源数据分析
   - 主要技术栈：Django + MongoDB + Elasticsearch + Python可视化
   - 核心特色：分布式存储系统、Python可视化技术栈、双数据库架构
   - 项目应该已100%完成，处于生产就绪状态

4. 掌握文档体系：
   - 查看最新的文档索引了解完整文档结构
   - 了解核心文档、技术文档、历史文档的分类
   - 确认重要配置文件和启动脚本的位置

5. 准备继续工作：
   - 确认项目当前完成状态和运行状况
   - 了解可以进行的后续工作(答辩准备、功能演示、技术优化等)
   - 准备好无缝继续之前的工作

请读取相关文档后，确认您已理解项目当前状态，并告知系统运行情况和可以进行的后续工作。
```

## 📋 指令特点

### **通用性设计**
- ✅ **文件名模糊匹配**: 使用通配符(*)匹配最新版本文档
- ✅ **时间无关性**: 不依赖特定日期，自动查找最新文档
- ✅ **状态自适应**: 根据实际运行状态进行检查
- ✅ **长期有效**: 可以在任何时候使用

### **智能查找机制**
- **最新状态文档**: 自动查找`CURRENT_PROJECT_STATUS_*.md`
- **课程对照文档**: 自动查找`项目开发进度对照文档_*.md`
- **文档索引**: 自动查找`📚_项目文档索引_*.md`
- **启动配置**: 自动查找`🚀_项目启动配置_*.md`

### **何时使用此指令**
1. **新开会话**: 由于对话长度限制需要新开会话
2. **跨时间工作**: 隔天、隔周继续项目工作
3. **环境切换**: 在不同设备或环境中工作
4. **协作交接**: 与其他开发者协作时
5. **系统重启**: 重新启动开发环境后
6. **状态同步**: 需要快速了解项目当前状态

使用此指令后，AI助手将：
1. ✅ 自动查找并读取最新的项目文档
2. ✅ 检查当前系统和服务运行状态
3. ✅ 理解项目完整背景和技术架构
4. ✅ 掌握文档体系和分类结构
5. ✅ 准备好继续任何后续工作

- **容错能力**: 即使某些文档缺失也能正常工作
- **扩展性**: 随着项目发展自动适应新的文档结构

## 🛠️ 故障排除

### **如果文档读取失败**
指令会自动尝试查找以下备选文档：
- `README.md` - 项目基本介绍
- `PROJECT_STATUS.md` - 旧版项目状态
- `DOCUMENTATION_INDEX.md` - 文档索引
- 项目根目录的所有`.md`文件

### **如果服务未运行**
指令会指导检查以下服务：
- Django服务器状态和端口占用
- MongoDB服务状态和连接
- Elasticsearch集群状态
- 系统资源使用情况

### **如果项目结构变化**
指令具有自适应能力：
- 自动发现新的文档结构
- 适应文档命名变化
- 识别新增的服务和配置
